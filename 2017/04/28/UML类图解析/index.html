UML类图示例

类图解析

“动物”矩形框代表一个类(Class)。

类图分三层：

  1. 第一层显示类的名称，如果为抽象类，则用斜体显示。
  2. 第二层表示类的特性，通常就是字段和属性
  3. 第三层表示类的操作，通常是方法或行为

关于符号，‘+’表示public，‘-’表示private，‘#’表示protected

接口图解析


接口图，与类图的区别是顶端有<< interface >>显示。

接口图同分为三层。(上图有问题，少了一层特性，即字段和属性)

  1. 第一层显示接口的名称
  2. 第二层表示接口的特性
  3. 第三层表示接口的方法

右边的接口表示方法为棒棒糖表示法。

右图表示“唐老鸭”这个类实现了“讲人话”这个接口。

类图常见的几种关系解析
泛化(Generalization)

泛化关系：是一种继承关系，它指定了子类如何特化父类的所有特征和行为。用空心三角形和实线来表示这种关系。

代码示例：

class Bird extends Animal{}
实现（Realization)

实现关系：是一种类与接口的关系，表示类是接口所有特征和行为的实现。用空心三角形和虚线来表示这种关系。

代码示例：

class Bird implements Animal{}
关联（Association）

关联关系：是一种拥有的关系,它使一个类知道另一个类的属性和方法。
关联可以是双向的，也可以是单向的。双向的关联可以有两个箭头或者没有箭头的实线，单向关联，带普通箭头的实线指向被拥有者。

代码示例：

class Penguin extends Bird{
    private Climate climate;
}
聚合（Aggregation）

聚合关系：是整体与部分的关系。聚合表示一种弱的拥有关系，体现的是A对象可以包含B对象，但是B对象不是A对象的一部分。用空心的**菱形和实线箭头**来表示这样关系(空心菱形指向整体，箭头指向部分)。

代码示例：

class WideGooseAggregate{
    private WideGoose[] arrayWideGoose;
}
组合(Composition)

组合关系：是整体与部分的关系。组合表示一种强的拥有关系，体现了严格的整体和部分的关系，部分和整体的生命周期一致。用实心的菱形和实线箭头来表示这样的关系(实心菱形指向整体，箭头指向部分)。

图中的数字表示，一个类的实例可以有多少个，图中表示1个鸟的类中有2个翅膀的实例。

代码示例：

class Bird {
    private Wing wing;
    public Bird(){
        wing = new wingl
    }
}
依赖(Dependency)

依赖关系：是一种使用的关系,所以要尽量不使用双向的互相依赖。用虚线箭头来表示这种关系。

代码示例：

abstract class Animal{
    public Metabolism(Oxygen oxygen,Water water){
    }
}

各种关系的强弱顺序：

泛化= 实现> 组合> 聚合> 关联> 依赖
